# 工具

# 多人协作

fda 不支持多人协作。而且因为数据的保存机制，还要监测是否有多人同时编辑同一页面的情况。

# case1. 本人在多个标签页打开了同一个页面

这种情况下，很可能出现这种场景：在 A 标签页打开的页面做了很多的编辑工作，然后切换到 B 标签页，B 标签页还停留在旧的状态，修改之后自动保存，把 A 标签页里的数据覆盖了。。。
检测到这种情况时，我们会在用户名旁添加一个警告标识：

同时在 A 和 B 标签页添加一个功能，切换到该标签页时，自动载入最新的存档。

# case2. 多人打开了同一个页面

除了数据的保存里介绍的弹窗外，页面底部还会有当前打开该页面的人员列表：

图中表示除了"我"之外，lijian3 在两个标签页都打开了该页面。

# 自动保存

我们每5秒会自动保存一次数据，不需要手动点击保存。

保存之后会记录保存的时间点。
除了自动保存，还可以手动点击保存区域，填写保存信息之后即可记录一份当前页面的快照。

和自动保存相比，手动保存更加精准。比如手动保存时填写，"测试新闻组件某一属性是否好用"，然后开始测试某一新闻组件，测试完毕之后想恢复之前的样子，相比于数量繁多的自动保存（因为每5秒一次），此时有个手动保存的记录会舒服很多。
保存的结果见保存记录。

# 页面信息


页面的英文名不可修改，其他的分别对应中文名和频道，点击修改即可，没什么可说的。

# 回退与前进


当回退或前进可用时，对应图标会变成高亮状态。点击之后即可触发回退或前进，也可以使用快捷键：
回退：Ctrl/⌘ + Z
前进：Shift + Ctrl/⌘ + Z

# 保存记录

自动保存和手动保存都会显示在这个工具里。

自动存档最多保存500条，每天凌晨 2:00 自动清理。

可以选择某一条存档，点击回滚。

# 尺寸设置


本工具修改的尺寸只用于编辑器的画布区域，不会影响预览和线上页面。
如果预设的几个尺寸不满足，还可以在下方填写像素值，手动指定宽度。
移动端则是定死的375px，不可修改。

# 出码

给开发用的，可以查看当前页面最终生成的代码，代码只读，虽然可以修改，但是不会影响最终代码。

本工具是阿里官方出的未开源的插件，修改起来太过费劲，想优化也无从下手，等后续看看会不会开源吧。

# 预览

点击 之后打开新的页面，可以快速预览最终的效果。
如果预览和最终效果不同，请反馈给我们:)。

# 发布

点击 来把页面发到生产环境或者测试环境。
发布时要写发布信息，用于记录本次发布的原因。
发布成功或者失败，都会在企业微信中发通知信息，注意查收。
鼠标悬浮右侧小箭头，可以选择把当前页面保存为模板，方便以后使用。
