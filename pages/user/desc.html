# 使用流程

搭建系统 —— fda 是一个可视化制作页面的工具，此类工具俗称"拼高达"。
使用fda可以大大提高我们产出页面的效率，好好学习使用，把主动权掌握在自己手中👏😅。

部分流程未开发完毕，后续可能有改动

# 1. 创建页面

后台管理页open in new window。
可以创建一个文件夹，也可以在现有文件夹中选择一个，在里面创建一个页面。
新建页面时有很多表单项，如图：

带点的是必填项，红点表示未填写，绿点表示已填写。
右侧 Tabs 里的是可选的配置项，一般情况下，只需要配置"基本"里的就够了。

属性说明
url页面的链接，填写 url 的同时，也设置了频道和项目名称
中文名称页面的中文名称，给人看的
端默认 PC 页面
选择模板模板里有很多预设的配置，如果有适合的模板，记得创建时在这里选择

# 2. 初识编辑器

为了保证使用的安全便捷，我们制作了很多工具。
现在不必一点点看这个图，等你对 fda 有了大致的印象，想深入了解时，可以按图索骥，在工具 中查阅。

# 3. 拖拽组件

组件是搭建页面的最小单位。
组件库中有很多组件，可以按组件库、分类、名称筛选。

# 浏览时

可以点击组件库面板右上角的图钉图标，总是展开组件库。
可以移到 上预览组件的大图。
表示该组件有多个皮肤。

# 拖拽时

拖拽的过程中，右侧会出现拖拽的预览结果：

当可选的层级太多时，可能放置位置不太准确，这时候一个多向右侧、上侧微调，大概率能找到合适的层级。
现在打开组件库，拖拽一个组件到页面中试试~

# 拖拽后

拖拽完成后，组件就会被放置到画布中，自动选中。
右侧设置器区域也出现了该组件的配置选项，修改配置项，画布中的组件也会同步更新。更详细的介绍，见编辑器区域。

# 3.1 重要的布局组件

布局组件，就是前端里的 div，文本里的段落。设置一个合适的结构，可以省很多事情。
太过重要以至于得一个个说明。

# 容器

容器就是一个盒子。
你可以把两个组件放在一个容器里方便管理；
也可以把好多组件放在一个容器里，容器设置主宽度，这样所有的组件就不需要再设置宽度，直接继承主宽度。
容器可以嵌套容器，适用于更复杂的情况。

属性说明
宽度容器的宽度，绝大部分的组件都没有写默认宽度，所以拖过来就能直接使用容器的宽度
是否居中容器本身是否居中

# 行

行里面只能放列。
行组件和列组件搭配，可以实现例如左文右图、分栏排版、左边一块右边一块等等常见布局。具体用法在列组件中介绍。

属性说明
宽度行的宽度
是否居中行本身是否居中
水平排列行里面的组件，在水平方向的对齐规则
垂直排列行里面的组件，在垂直方向的对齐规则

# 列

列只能被放在行里。

  1. 先拖一个行组件
  2. 在行组件中拖若干个列组件
  3. 在列组件中拖拽业务组件
属性说明
宽度列的宽度，百分比：父容器（也就是列）宽度的百分比；像素：固定的像素值；flex：见下。

flex 介绍：
文本框里应该写纯数字，表示与其他 flex 值的比例。举个例子：

  1. 行里面有一个列组件，flex 写任意值都是父容器宽度的 100%；
  2. 行里面有两个列组件，A 设置为 300px，B 设置 flex 为任意值，那么B的宽度则是父容器宽度减去 300px，因为没有其他的 flex 的值了；
  3. 行里面有三个列组件，A 设置为 300px，B 设置 flex 为 1，C 设置 flex 为 3，那么 B 和 C 的宽度是父容器宽度减去 300px 之后的 1:3。

# 通栏

通栏和容器的功能一样，只是多了一个隐藏的功能。

在编辑器内使用，和容器一致。
在预览页、测试/生产环境的可视化编辑查看时，会在通栏的右上方添加一个小按钮，表示该区域是否隐藏。

鼠标放置在按钮上可查看该通栏覆盖的区域。
点击按钮可切换显隐状态。
预览页、测试环境、生产环境三者共享隐藏状态。

# 4. 了解区块

考虑这种场景，一个很复杂的组件，吭哧吭哧配置完了，觉得很好用，下次还想用，甚至想安利出去让别人也用。
或者某一部分的页面里有容器、行、列、新闻列表、轮播图、视频等等，这次配置完了，想保存一下，下次使用的时候换个皮就行了。

这就是区块存在的意义。

# 4.1 区块的产生


选中一个组件，点击 + 号，填写区块的名称和描述（图片自动生成），再点击保存即可保存区块。
值得注意的是，生成的区块所有人可见，你可以使用别人的区块，别人也可以使用你的区块。

  • 为了防止数据被误用，保存区块时我们会去掉区块里的碎片信息。

# 4.2 区块的使用

# 区块列表

最常用的就是左侧的区块列表，你可以像组件一样筛选、拖拽它们。

# 社区方案

比如你正在编辑一个组件 Nav_1，此时选择右侧的社区方案，会展示所有 Nav_1 的区块，点击任一区块，则可以把这个区块的配置项应用到组件中。

# 5. 数据的保存

我们每5秒会自动保存一次数据，不需要手动点击保存。
退出页面时也会自动保存，可放心关闭页面。

注意：
fda 不支持多人协作，且因为自动保存的机制，多人同时编辑同一页面时，双方会不断覆盖对方的记录，所以这种情况我们会弹出一个不自动关闭的弹窗，提示尽快和对方沟通。

关于自动保存的详情，可以查看工具-自动保存。

# 6. 发布

页面完成之后，可以点击右上角的 来把页面发到生产环境或者测试环境。
发布时可以选择填写发布信息，用于记录本次发布的原因。
发布成功或者失败，都会在企业微信中发通知信息，注意查收。
