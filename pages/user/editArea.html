# 编辑器区域

编辑器是 fda 的核心，用户在这里设置组件的属性。
分为基本、通用和社区方案三个 tab，其中社区方案已经在使用流程-了解区块中做了介绍，这里只介绍基本和通用。

# 基本

基本里有很多设置器，大部分都很简单，这里介绍几个特殊的、复杂的设置器。

# 数据

数据设置器chipSetter是最复杂也是最重要的设置器，用于设置组件内的数据。
数据来源可以是碎片、精品池、接口，还能设置打底数据。

# 碎片

组件创建时，会根据示例碎片自动生成新的碎片，不必再手动创建碎片。
在文本框内输入碎片id来查找碎片，点击碎片内容来启用、清除或者编辑碎片。

  • 由于推荐位碎片、精品池、动态碎片三者返回的数据格式一致，所以可以互相切换，切换时将清除碎片；
# 接口

点击这里来切换到接口面板：

直接在文本框内输入接口地址即可。

  • 仅支持GET请求，并且要求接口支持跨域。
# 打底数据

点击这里来切换到打底数据的面板：

  • 打底数据只支持精品池、动态碎片和接口；
  • 干涉数据只支持推荐位；
  • 在推荐位碎片设置里填写固定位置来固定碎片数据的位置。

# 皮肤

皮肤是我们提前准备好的、推荐的样式方案。
可以在轮播图里浏览、选择方案。左下角列出了启用该方案可能会影响的属性。

# 通用

通用里的样式设置更偏向于技术人员，我们会尽量做得更加容易理解。。。
每个组件都有通用样式，但是这里的样式只影响顶层容器。比如有一个新闻列表的盒子，里面放着好多新闻，通用样式只影响盒子，不影响盒子里的新闻（除非有CSS属性的继承）。
