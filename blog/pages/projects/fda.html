搭建系统 - fda ​

搭建系统见的也是比较多了，我认为每个需要固定产出 web 页面的公司都需要一个搭建系统来提升效率。

fda 对我提升巨大，因为从调研到实际线上使用都是我主导开发的。
在这个庞然大物里，我负责了：

  • fda_editor(低代码编辑器)
    用户操作组件、编辑布局、生产页面的主要工具。
  • basic、business 等组件物料库
    放置各种组件的物料库。除了实现组件的功能，还需考虑各个 props 的配置项
  • fda_pages(页面后台)
    生产出的页面的管理后台，也是创建编辑页面的入口
  • fda_server
    各种 curd 操作的简单接口
  • fda_tool 普遍用于组件物料库中的公共方法、ts 规范等，封装成了一个 npm 包

当然也有一些是可爱的同事们参与完成的。比如页面完成后的出码、检测多人协作的websocket 服务。

下面只介绍最重要的 fda_editor(低代码编辑器)。

特色功能介绍 ​

1. 重要的布局组件 ​

布局组件，就是前端里的 div，文本里的段落。设置一个合适的结构，可以省很多事情。
我设计了三种布局组件。

容器 ​

容器就是一个盒子。
你可以把两个组件放在一个容器里方便管理；
也可以把好多组件放在一个容器里，容器设置主宽度，这样所有的组件就不需要再设置宽度，直接继承主宽度。
容器可以嵌套容器，适用于更复杂的情况。

属性 说明
宽度 容器的宽度，绝大部分的组件都没有写默认宽度，所以拖过来就能直接使用容器的宽度
是否居中 容器本身是否居中

行 ​

行里面只能放列。
行组件和列组件搭配，可以实现例如左文右图、分栏排版、左边一块右边一块等等常见布局。具体用法在列组件中介绍。

属性 说明
宽度 行的宽度
是否居中 行本身是否居中
水平排列 行里面的组件，在水平方向的对齐规则
垂直排列 行里面的组件，在垂直方向的对齐规则

列 ​

列只能被放在行里。

  1. 先拖一个行组件
  2. 在行组件中拖若干个列组件
  3. 在列组件中拖拽业务组件
属性 说明
宽度 列的宽度，百分比：父容器（也就是列）宽度的百分比；像素：固定的像素值；flex：见下。

flex 介绍：
文本框里应该写纯数字，表示与其他 flex 值的比例。举个例子：

  1. 行里面有一个列组件，flex 写任意值都是父容器宽度的 100%；
  2. 行里面有两个列组件，A 设置为 300px，B 设置 flex 为任意值，那么B的宽度则是父容器宽度减去 300px，因为没有其他的 flex 的值了；
  3. 行里面有三个列组件，A 设置为 300px，B 设置 flex 为 1，C 设置 flex 为 3，那么 B 和 C 的宽度是父容器宽度减去 300px 之后的 1:3。

2. 区块 - fda 里的“组件” ​

考虑这种场景，一个很复杂的组件，吭哧吭哧配置完了，觉得很好用，下次还想用，甚至想安利出去让别人也用。
或者某一部分的页面里有容器、行、列、新闻列表、轮播图、视频等等，这次配置完了，想保存一下，下次使用的时候换个皮就行了。

这就是区块存在的意义。

区块的产生 ​


选中一个组件，点击 + 号，填写区块的名称和描述（图片自动生成），再点击保存即可保存区块。
值得注意的是，生成的区块所有人可见，你可以使用别人的区块，别人也可以使用你的区块。

区块的使用 ​

  1. 区块列表：所有区块的列表，你可以像组件一样筛选、拖拽它们。
  2. 社区方案：
    比如你正在编辑一个组件 Nav_1，此时选择右侧的社区方案，会展示所有 Nav_1 的区块，点击任一区块，则可以把这个区块的配置项应用到组件中。

3. 皮肤 ​

皮肤是我们提前准备好的、推荐的样式方案。
可以在轮播图里浏览、选择方案。左下角列出了启用该方案可能会影响的属性。

4. 自动保存 ​

我们每5秒会自动保存一次数据，不需要手动点击保存。

保存之后会记录保存的时间点。
除了自动保存，还可以手动点击保存区域，填写保存信息之后即可记录一份当前页面的快照。

和自动保存相比，手动保存更加精准。比如手动保存时填写，"测试新闻组件某一属性是否好用"，然后开始测试某一新闻组件，测试完毕之后想恢复之前的样子，相比于数量繁多的自动保存（因为每5秒一次），此时有个手动保存的记录会舒服很多。

5. 保存记录 ​

自动保存和手动保存都会显示在这个工具里。

自动存档最多保存500条，每天凌晨 2:00 自动清理。

可以选择某一条存档，点击回滚。

6. 不支持多人协作，但... ​

fda 不支持多人协作。但为了数据能正确地保存，还要监测是否有多人同时编辑同一页面的情况。

case1. 本人在多个标签页打开了同一个页面 ​

这种情况下，很可能出现这种场景：在 A 标签页打开的页面做了很多的编辑工作，然后切换到 B 标签页，B 标签页还停留在旧的状态，修改之后自动保存，把 A 标签页里的数据覆盖了。。。

同时在 A 和 B 标签页添加一个功能，切换到该标签页时，自动载入最新的存档。
